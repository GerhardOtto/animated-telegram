# EPI-USE Labs - Technical Challenge

> I decided to use this repo as general example of how I would approach a project.

## Development Workflow

### Branching Strategy

- Branch off `master` for every fix or feature — never commit directly to `master`
- Keep branches short-lived and focused on a single concern
- Use Release Please for automated version management of the project through conventional commit messages.

---

### Conventional Commits

All commits follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
<type>(<optional scope>): <description>
```
